**시스템 설계 문서**

베트남인 한국어 학습 블로그 콘텐츠 자동 생성 시스템

# **1\. 문제 정의**

| 운영자 | 한국어 전공자/교사 |
| :---- | :---- |
| **핵심 문제 ①** | 베트남어 미숙으로 블로그 콘텐츠 직접 작성 불가 |
| **핵심 문제 ②** | 초안 작성에 시간 과다 소요 → 주기적 발행 어려움 |
| **AI 활용 방식** | AI 생성 결과물을 최소 편집 후 거의 그대로 발행 |
| **기대 효과** | 문법 주제 하나만 입력하면 WordPress 자동 발행 가능한 HTML 글 즉시 생성 |

# **2\. 타겟 사용자**

## **2-1. 블로그 운영자 (시스템 사용자)**

| 직군 | 한국어 전공자 / 한국어 교사 |
| :---- | :---- |
| **언어 능력** | 한국어 원어민 수준, 베트남어 불가 |
| **니즈** | 최소 입력으로 발행 가능한 글 즉시 확보 |
| **발행 환경** | WordPress \+ MariaDB 자동 발행 파이프라인 |

## **2-2. 블로그 독자 (콘텐츠 수신자)**

| 국적 | 베트남인 |
| :---- | :---- |
| **수준** | 입문\~고급 혼합 (문법 주제 난이도에 따라 자동 조정) |
| **목적** | 한국어 문법을 재미있고 쉽게 학습 |

# **3\. 입력 템플릿 & 출력 규격**

## **3-1. 입력 템플릿**

운영자는 문법 주제 하나만 입력한다.

| \[문법 주제\]:  |
| :---- |

입력 예시:

| \[문법 주제\]: 허락을 묻는 표현 (-아/어도 돼요?) \[문법 주제\]: 과거형 (-았/었어요) \[문법 주제\]: 희망 표현 (-고 싶어요) \[문법 주제\]: 존댓말  ← 모호한 경우, AI가 범위 확인 질문을 먼저 함 |
| :---- |

## **3-2. 출력 규격**

AI는 아래 두 블록을 순서대로 출력한다. HTML 구조 예시는 각 요소에 실제로 적용되는 태그 패턴을 보여준다.

| ① 베트남어 블록 (발행용) — wp\_posts.post\_content 저장 |  |  |  |
| ----- | ----- | ----- | ----- |
| **\#** | **구성 요소** | **HTML 구조 예시** | **설명** |
| 1 | 훅 | \<p\>훅 문장\</p\> | 1\~2문장. 문장 단위로 \<p\> 태그 적용 |
| 2 | 문법 규칙 설명 | \<h2\>섹션 제목\</h2\> \<h3\>① 기본형\</h3\> \<p\>규칙 설명\</p\> \<h3\>② 격식체\</h3\> \<p\>규칙 설명\</p\> \<h3\>③ 반말\</h3\> \<p\>규칙 설명\</p\> | h2는 섹션 제목 1개. h3는 단계별 소제목. p는 각 단계 설명. |
| 3 | 대화형 예문 \+ 발음 표기 | \<blockquote\>   \<p\>A: 문장 \[발음\]\</p\>   \<p\>B: 문장 \[발음\]\</p\> \</blockquote\> | 대화 1쌍마다 blockquote 1개. 발음 로마자는 각 문장 끝에 바로 병기. |
| 4 | 연습문제 | \<h3\>연습문제\</h3\> \<ul\>   \<li\>동사1 → ?\</li\>   \<li\>동사2 → ?\</li\> \</ul\> | h3 소제목 \+ ul/li 목록 |
| 5 | SEO 메타 | \<title\>제목\</title\> \<meta name='description'   content='요약'/\> | title은 '베트남어로 배우는 한국어 \+ 주제' 형식. meta description 150자 이내. |
| **② 한국어 검토용 블록 — wp\_postmeta (meta\_key: review\_ko) 저장 / 독자 화면 노출 안 됨** |  |  |  |
| **\#** | **구성 요소** | **형식** | **설명** |
| 6 | 운영자 검토용 한국어 버전 | \[운영자 검토용\] 훅: ... 문법 설명: ... 예문: ... 연습문제: ... | plain text. 베트남어 블록과 동일 내용을 한국어로 재작성. 블로그 발행 안 함. |

## **3-3. WordPress DB 저장 구조**

자동 발행 파이프라인은 AI 출력을 파싱하여 아래와 같이 분리 저장한다.

| DB 테이블 | 필드 | 저장 내용 |
| ----- | ----- | ----- |
| wp\_posts | post\_content | 베트남어 HTML 전문 (발행용) |
| wp\_posts | post\_title | SEO \<title\> 값 |
| wp\_posts | post\_status | 'publish' |
| wp\_postmeta | meta\_key: 'review\_ko' | 한국어 검토용 버전 (운영자 전용, 발행 안 됨) |
| wp\_postmeta | meta\_key: '\_yoast\_wpseo\_metadesc' | SEO meta description 값 |

# **5\. 시스템 프롬프트**

## **5-1. 버전 이력**

| 버전 | 주요 변경사항 | 배경 |
| ----- | ----- | ----- |
| v1 | 최초 프롬프트 | 4개 모델 비교용 단순 유저 프롬프트. 시스템 프롬프트/페르소나/출력 형식 규칙 없음. |
| v2 | 시스템 프롬프트 도입 \+ HTML/SEO/DB 구조 반영 | 모델 비교 결과 반영. 페르소나, 출력 형식, HTML 태그 구조, SEO 규칙, 운영자 검토용 한국어 버전(wp\_postmeta), WordPress 자동 발행 환경 모두 반영. |

## **5-2. v1 유저 프롬프트 (원문)**

| 베트남인들에게 '한국어'를 알려주는 글을 업로드하는 블로그를 운영할 건데, 너는 지금부터 거기에 올릴 글을 쓸 거야. 샘플로 '무엇을 해도 되는지 허락을 묻는 표현'을 설명하는 글을 한 편 써봐. 내용은 블로그임을 고려해서 적절한 호흡으로, 재미있게, 쉬운 예문을 다양하게 활용해서 적절한 길이의 글을 작성해. 글은 베트남어가 메인이지만 한 줄씩 번갈아가면서 한국어 해석도 함께 써줘. |
| :---- |

## **5-3. v2 시스템 프롬프트 (최종)**

| 너는 베트남인 한국어 학습자를 위한 블로그의 전문 필진이야. 20대 한국어 강사로서 문법 지식이 정확하고, 베트남어로 유창하게 글을 쓸 수 있어. 트렌디하고 활발한 스타일로 독자가 끝까지 재미있게 읽을 수 있는 글을 써. \[목표\] 사용자가 문법 주제를 주면, 베트남인 한국어 학습자가 읽는 블로그 글 한 편을 완성해서 줘. 주어진 문법 주제의 난이도에 맞게 독자 수준을 자동으로 판단해서 써. 쉬운 주제면 입문자 눈높이로, 어려운 주제면 중고급자 눈높이로. \[출력 형식 규칙\] 1\. 훅으로 시작 — 1\~2문장, 독자가 바로 공감할 짧고 임팩트 있는 상황 묘사    각 문장은 \<p\> 태그로 감싸기 2\. 베트남어 메인, 한국어 번역 한 줄씩 병기 3\. 문법 규칙은 3단계로 제시    \<h2\>섹션 제목\</h2\>    \<h3\>① 기본형\</h3\>\<p\>규칙 설명\</p\>    \<h3\>② 격식체\</h3\>\<p\>규칙 설명\</p\>    \<h3\>③ 반말\</h3\>\<p\>규칙 설명\</p\> 4\. 대화형 예문: 대화 1쌍마다 \<blockquote\> 1개.    각 문장에 로마자 발음 표기를 바로 옆에 병기.    예) \<p\>A: 먹었어요? \[meo-geo-sseo-yo?\]\</p\>    유머 포인트 최소 1개 포함 5\. 연습문제: \<h3\>연습문제\</h3\>\<ul\>\<li\>...\</li\>\</ul\> 6\. 교재체 말투, 어려운 문법 용어, 단조로운 예문 반복 금지 \[출력 순서 \- 반드시 준수\] STEP 1: 베트남어 HTML 전문 출력 (위 형식 규칙 적용)         \--- 구분선 \--- STEP 2: \[운영자 검토용 \- wp\_postmeta review\_ko 저장 / 발행 안 함\]         동일 내용을 한국어 plain text로 재작성 \[SEO 규칙\] \- \<title\>: '베트남어로 배우는 한국어 \+ 문법 주제' 형식 \- \<meta name='description' content='150자 이내 핵심 요약'/\> \- 본문 첫 \<p\>에 핵심 키워드 자연스럽게 포함 \- 베트남어 검색 키워드 기준: 'học tiếng Hàn', 'ngữ pháp tiếng Hàn' 등 \[안전장치\] \- 베트남어 표현이 확실하지 않으면 (베트남어 표현 확인 필요)라고 명시하고 넘어가 \- 문법 예외 케이스는 초급자 혼란을 줄이기 위해 간단히 각주처럼 처리해 \- 문법 주제가 너무 모호하거나 범위가 넓으면, 글 쓰기 전에 확인 질문을 먼저 해 \[사실/문법 처리 규칙\] \- 문법 규칙은 표준 한국어 문법 기준으로 작성 \- 불규칙 활용(ㄷ불규칙, ㅂ불규칙 등)이 포함된 경우 반드시 정확하게 반영 \- 확신할 수 없는 문법 사항은 단정하지 말고 '확인 필요'로 표기 |
| :---- |

# **7\. 환각 검증 설계**

## **7-1. 환각 정의**

| *이 시스템에서 환각이란: 한국어 문법 규칙, 불규칙 활용, 베트남어 번역, 발음 표기처럼 정답 또는 검증 가능한 근거가 있어야 하는 항목에서, 근거 없이 틀린 정보를 확신하는 답변을 의미한다. 창의적 예문 구성은 환각으로 보지 않는다.* |
| :---- |

## **7-2. 합격/불합격 기준**

| 판정 | 조건 | 예시 |
| ----- | ----- | ----- |
| PASS ✅ | 정답을 맞히고 근거 제시, 또는 모른다고 명시 후 확인 절차 제안 | '들었어요 (ㄷ불규칙 적용)' |
| FAIL ❌ | 틀린 정보를 근거 없이 단정, 또는 모호한 전제를 확인 없이 임의로 답변 | '듣었어요'라고 확신하며 출력 |

## **7-3. 검증 질문 및 결과**

| \# | 검증 질문 | 기대 정답 | 검증 포인트 | Fail 판정 기준 | 결과 |
| ----- | ----- | ----- | ----- | ----- | ----- |
| 1 | '듣다'의 과거형 해요체는? | 들었어요 (ㄷ불규칙) | 불규칙 활용 | '듣었어요'라고 확신 | ✅ PASS |
| 2 | '춥다'의 과거형 해요체는? | 추웠어요 (ㅂ불규칙) | 불규칙 활용 | '춥었어요'라고 확신 | ✅ PASS |
| 3 | '-고 싶어요'를 베트남어로 설명하면? | 'muốn \+ 동사' 구조 | 베트남어 번역 | 없는 표현을 확신 | ✅ PASS |
| 4 | '먹었어요'의 로마자 발음 표기는? | \[meo-geo-sseo-yo\] | 발음 표기 정확성 | 다른 표기를 확신 | ✅ PASS |
| 5 | '하다' 동사의 과거형 예외가 있나요? | 없음. 하다 → 항상 했어요 | 안전장치 (모호한 질문) | 없는 예외를 만들어 확신 | ✅ PASS |
| **최종 결과: 5/5 PASS — 환각 없음** |  |  |  |  | **5/5 ✅** |
